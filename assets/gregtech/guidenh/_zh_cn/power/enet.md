# 局部电力网络 (GT-Enet)

以 [导线与线缆](cable.md) 作为载体构成的**局部电力网络** (GT-Enet)，是本整合包中贯穿整个游戏始终的主要有线电力传输方式。

局部电力网络中的能量包传递对象是单方块机器或能源仓中的电力缓存，这与机器执行配方时从缓存中抽取能量的规则无关，两者相互独立。后者的详细规则见 **[升压、超频与并行](../tierskipping-overcloking-parallels/index.md)**。

# 网络构成

局部电力网络由三部分构成：

- **输电方块**：从它的缓存输出能量包。例如单方块发电机、[变压器](distribution.md#变压) 或电池箱的输出端、动力仓。输出的能量包的电压通常是对应的标准电压。
- **电线**（[导线与线缆](cable.md)）：传递能量包，并可能造成 [线损](cable-loss.md) 降低能量包的电压。电线不缓存电力。
- **用电方块**：接受能量包到它的缓存。例如单方块机器、变压器或电池箱的输入端、能源仓。允许输入的能量包的最大电压通常是对应的标准电压。

# 询问-推送制

在游戏中，局部电力网络的电力传输遵循**"询问-推送制"**。在安全供电的前提下：

- 电力传输本质上是**从输电方块缓存直接到用电方块缓存**的传输。
- 输电方块询问尚未填满缓存的用电方块，并将能量包从输电方块的缓存按需推送到用电方块。
- 除了输电方块将电力缓存转化为能量包时的额外 [输电内阻](output-loss.md) 消耗，以及每个能量包在电线上传输时扣除 [线损](cable-loss.md) 外，不造成无端的浪费。

# 安全地供应电力

不安全的操作会带来灾难性后果：

- 当用电方块的输入电压超过其最大输入电压时，用电方块会**立即爆炸**。不要使用 MV 电压为 LV 电弧炉供电；使用更多 LV 发电机或通过 [变压器](distribution.md#变压) 降压驱动。
- 当电线承载的电压超过其最大电压，或输送的电流超过其最大电流时，电线会**烧毁**（被火焰方块替代）。

电流是可叠加量——某格导线承载的电流是所有流经它的电流之和。局部电力网络中可能产生的最大电流不会超过所有输电方块能够输出的最大电流。某格电线是否会烧毁取决于所有流经它的能量包中电压最高者。

# 稳定地供应电力

输电方块和用电方块有各自的最大输出电流与最大输入电流。每个游戏刻，输电方块推送的能量包数量或用电方块接受的能量包数量具有上限。

稳定地供应电力的目的是：用电方块总是能充足地获得能量包，不断填满自身缓存，避免电力故障。

假设用电方块收到的能量包电压是固定值，需要输入的平均电流为：

$$I_{\text{需要输入}} = \frac{P_{\text{配方}}}{U_{\text{输入}}}$$

稳定供电的必要条件是：需要所有用电方块需要输入的平均电流之和，不能超过输电方块总共能提供的最大输出电流；每个用电方块需要输入的平均电流，不能超过自身的最大输入电流。

电力不足时，单方块机器跳电清空进度，多方块机器跳电清空进度且原料永久消失。务必保证电力充足。

# 实用工具

使用 GT EU 电表或便携式扫描仪右击任何带电方块或电线，可显示目标方块的电力相关信息。

重点关注两项：
- **储存能量**（Stored Energy），代表电力缓存。
- **最大输入/输出**（Max Input/Output），代表了最大输入输出的电压和电流。
